3. V-Mini làm đúng những gì

Token-first architecture

Mọi component đều bind vào variables. Đổi theme = tự động đổi toàn bộ UI. Không hardcode hex ở bất kỳ đâu.

3-tier token structure

Primitive (raw colors) → Semantic (intent: success, danger) → Component (button-bg, input-border). AI đọc được intent, không chỉ thấy hex.

Documentation trong Figma

Mỗi component có description, usage guidelines, do/don't ngay trong file. Designer mới và AI đều hiểu cách dùng mà không cần hỏi ai.

Naming convention nhất quán

Mọi token, component, variant đều theo cùng convention. AI parse được, search được, map được về code không sai.

4. VSP mạnh ở đâu

Component library rộng nhất

692 components, 103 component sets — phủ gần hết mọi use case của app fintech. V-Mini không có độ sâu này.

Color token file đã mature

288 tokens, 2 collections (Primitive + Semantic), đã có Light/Dark modes. Nền tảng token đã sẵn — chỉ cần apply.

Code tokens hoàn chỉnh

globals.css có 200+ CSS variables, đã production-ready. Phần code đi trước Figma.

Visual quality cao

Design language rõ ràng: minimal, premium, nhất quán. Chỉ thiếu hệ thống hóa, không thiếu thẩm mỹ.

5. Gap cần xóa bỏ (ưu tiên cao → thấp)

1. Variables chưa apply vào components

Đây là gap lớn nhất. VSP có 288 color tokens nhưng 692 components vẫn dùng hardcoded hex. Khi AI đọc component, nó thấy #1f1f1f thay vì Background/Brand. Không hiểu được intent.

CriticalToken system +3AI-friendliness +2
2. Không có Code Connect

Figma và code là 2 thế giới riêng. Developer phải đoán component nào trong code tương ứng với component nào trong Figma. V-Mini có mapping rõ ràng.

CriticalCode↔Figma +5
3. Documentation gần như không có

55 pages trong Core Components file nhưng không có trang nào là documentation. Không có usage guidelines, do/don't, hay token reference. Designer mới = lạc.

CriticalDocumentation +5
4. Theme chưa hoạt động end-to-end

Color file có Light/Dark modes nhưng components không subscribe. Chuyển Dark mode = không có gì thay đổi.

ImportantTheme +3
5. Thiếu Spacing & Radius tokens

Code có px-[22px], rounded-[28px] nhưng Figma không có spacing/radius variables. Mỗi designer tự nhập số — dễ sai.

ImportantConsistency
6. Typography file thiếu documentation

Variables tồn tại nhưng không có visual preview, không có usage mapping. Không ai biết text-lg là dùng cho gì.

Nice to have

Kết luận

VSP không thua V-Mini về visual hay component depth — thậm chí mạnh hơn. Điểm yếu nằm hoàn toàn ở hệ thống hóa: tokens chưa apply, docs chưa viết, code chưa connect. Đây là những việc có thể làm trong 1-2 tuần với AI hỗ trợ, và sẽ nâng VSP từ 5.6 lên 8.5+/10.

VSP có nguyên liệu tốt. Chỉ cần hệ thống hóa lại.
